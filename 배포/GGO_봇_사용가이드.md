## 2. 봇 3종 세트 개요

봇 3종 세트는 **리더 1명 + 봇 N명**이 파티를 이뤄 던전을 자동으로 공략하는 시스템입니다.

```
┌─────────────────────────────────────────────┐
│              리더 컨트롤 패널                 │
│  (리더 캐릭터가 GUI로 봇 전체를 지휘)         │
└──────────────┬──────────────────────────────┘
               │ 채팅 명령어로 통신
       ┌───────┴────────┐
       ▼                ▼
  쓰로잉봇            바드봇
  (자동 전투)     (버프/힐/서포트)
```

**리더와 봇의 역할 분리**
- **리더**: 리더 컨트롤 스크립트를 켜고, GUI 버튼으로 팀 전체를 지휘합니다.
- **쓰로잉봇**: 리더를 따라다니며 원거리 자동 전투를 수행합니다.
- **바드봇**: 리더를 따라다니며 디스코던스/프로보케이션 + 힐을 담당합니다.

## 3. 공통 초기 설정 (첫 실행)

### 사전 준비

- **모든 봇 계정**: 백팩에 **룬북** 1개 준비 (파티 전용)
- **쓰로잉봇/바드봇**: 백팩에 **먹이** 준비 (펫이 있는 경우)
- **RazorEnhanced Vendor Sell 에이전트**: 목록 이름 `ps` 로 파워스크롤 등록
- **RazorEnhanced Organizer 에이전트**: 목록 이름 `trash` 로 버릴 아이템 등록
- **RazorEnhanced Dress 에이전트**: 쓰로잉봇은 `th`, 바드봇은 `luck` 으로 부활 장비 등록

### 스크립트 설정값 수정 (최초 1회)

각 봇 스크립트 상단 **"사용자 설정" 구역**만 수정하세요.

**쓰로잉봇 주요 설정값**
```python
dress_name            = "th"     # Dress 에이전트 이름
NPC_SELL              = 0x????   # 파워스크롤 판매 NPC 시리얼
TRASH_ORGANIZER_NAME  = "trash"  # Organizer 에이전트 이름
```

**바드봇 주요 설정값**
```python
musicbuff    = 1   # 1: 프롭버프(딜), 0: 피스버프(생존)
protection   = 1   # 1: Protection 버프 자동 유지
dress_name   = "luck"
NPC_SELL     = 0x????
```

### 첫 실행 시 설정 마법사

스크립트를 처음 실행하거나 파티 구성이 바뀌면 **설정 마법사**가 자동으로 시작됩니다.

```
1. "리더를 선택하세요" → 리더 캐릭터를 클릭
2. "룬북을 선택하세요" → 백팩의 룬북을 클릭
3. "파티원 선택 (우클릭 종료)" → 파티원들을 순서대로 클릭 → 우클릭으로 완료
4. "펫 선택 (우클릭 종료)" → 펫을 클릭 → 우클릭으로 완료 (없으면 바로 우클릭)
```

> 설정은 `%APPDATA%\GGO_Project\` 에 자동 저장되어 다음 실행 시 자동 로드됩니다.

## 4. GGO 리더 컨트롤 v1.1

리더 캐릭터가 사용하는 **팀 지휘 컨트롤 패널**입니다.
스크립트를 실행하면 화면에 다중 페이지 검프(GUI)가 열립니다.

### 검프 페이지 구성

| 페이지 | 내용 |
|--------|------|
| **메인** | 전투 모드, 에너미, 타겟 지정, 일시정지, 와파 등 |
| **펫&이동** | 펫 소환, 먹이, 게이트 지정, 헬프탈출 |
| **정리용** | 파스판매, 상하차, 파스/분홍/해골 정리 |
| **직업특화** | 바드 명령, 골드전송, 뱀파폼, 비행 |
| **보스설정** | 보스 알림 ON/OFF, 보스 루팅 ON/OFF |

### 메인 페이지 버튼 설명

| 버튼 | 기능 |
|------|------|
| **아머** | 쓰로잉봇의 Armor Ignore 상시 사용 ON/OFF |
| **선공** | 쓰로잉봇 선공 모드 (전투 개시 후 공격) |
| **챔프모드 ▶** | 특정 봇의 챔프 앵커 구역 지정 |
| **일반모드** | 모든 봇을 기본 전투 모드로 복귀 |
| **맥스모드** | 원거리 최대 유지 모드 |
| **에너미 ON/OFF** | Enemy of One 버프 토글 |
| **팔로잉 ON/OFF** | 팔로잉 모드 토글 (전투 억제, 리더만 따라감) |
| **타겟지정** | 적을 클릭하면 모든 봇이 집중 공격 |
| **와파 ON/OFF** | 자동 Wildfire 시전 ON/OFF (지점 클릭으로 설정) |
| **일시정지 / 재개** | 봇 전체 일시정지 / 재개 |

### 정리용 페이지 설명

| 버튼 | 기능 |
|------|------|
| **파스판매** | 봇들이 파워스크롤 NPC에게 자동 판매 |
| **상하차** | 봇들의 아이템을 지정 컨테이너로 이동 |
| **파스정리** | 봇들이 파워스크롤을 지정 컨테이너로 정리 |
| **분홍정리** | 봇들이 분홍 아이템을 지정 컨테이너로 정리 |
| **해골정리** | 봇들이 해골 아이템을 지정 위치로 정리 |
| **리더파스** | 리더 본인이 직접 파스를 컨테이너에 넣음 |
| **리더상하차** | 리더 본인의 아이템을 컨테이너로 이동 |
| **리더판매** | 리더가 직접 파워스크롤 판매 |

### 보스 알림 기능

주변 **25타일** 이내에서 보스 몬스터가 탐지되면:
- 화면에 빨간 헤드메시지로 알림
- Discord Webhook으로 알림 전송 (설정 시)

**감지되는 보스**: Rikktor, Barracoon, Neira, Semidar, Mephitis, Lord Oaks, Queen Silvani

Discord 알림을 받으려면 스크립트 상단에 웹훅 URL을 입력하세요:
```python
DISCORD_WEBHOOK_URL = "https://discord.com/api/webhooks/..."
```

## 5. GGO 쓰로잉봇 v1.1

**가고일 팔라딘 투척 전사** 전용 자동 전투 봇입니다.

### 검프 버튼 설명

| 버튼 | 기능 |
|------|------|
| **시작 / 정지** | 봇 작동 시작 또는 일시정지 |
| **재설정** | 리더/룬북/파티 설정 다시 입력 |
| **아머 On/Off** | Armor Ignore 상시 사용 모드 토글 |
| **먹이지정** | 펫 먹이 아이템 직접 선택 |
| **송금 On/Off** | `!송금` 명령 수신 시 골드 수집 여부 |
| **종료** | 스크립트 종료 |

### 자동 전투 동작 방식

```
시작 시: 뱀파이어폼 자동 시전 → 비행 활성화
         ↓
전투 감지: 리더 주변 14타일 내 적 탐지
         ↓
카이팅: 적이 4타일 이내 접근 시 반대 방향으로 이동 (무빙샷)
         ↓
버프 관리: Consecrate Weapon → Divine Fury (스태미너 부족 시)
         ↓
저주 감지: 최대 스태미너 5 이상 감소 시 Remove Curse 자동 시전
```

### 전투 모드

| 모드 | 조건 | 설명 |
|------|------|------|
| **일반** | 기본값 | 전투 중인 적만 공격 |
| **선공 (`!선공`)** | 리더 명령 | 전투 개시 안 해도 선공 |
| **맥스 (`!max`)** | 리더 명령 | 12타일 거리 유지하며 원거리 공격 |
| **챔프 (`!챔프 [이름]`)** | 리더 명령 | 지정 좌표 반경에서 독립 순찰 전투 |

### 리더 소실 시 동작

리더 시야(30타일) 밖으로 벗어나면 마지막 이동 방향으로 6타일 추격 후 대기합니다.

### 펫 관리

- 펫 HP 55% 미만 → `all follow me` 발령 (펫이 리더 뒤로 빠짐)
- 펫 HP 70% 이상 회복 → `all guard me` 복귀
- 최대 3회 재시도 (4초 간격)

## 6. GGO 바드봇 v1.0

**바드 스킬 기반 서포터** 자동화 봇입니다. 힐, 버프, 디스코던스/프로보케이션을 자동으로 수행합니다.

### 검프 버튼 설명

| 버튼 | 기능 |
|------|------|
| **시작 / 정지** | 봇 작동 시작 또는 일시정지 |
| **재설정** | 리더/룬북/파티 설정 다시 입력 |
| **디코 / 프로보 모드** | 바드 스킬 모드 전환 |
| **먹이 재설정** | 펫 먹이 아이템 직접 선택 |
| **송금 On/Off** | 골드 수집 여부 |
| **종료** | 스크립트 종료 |

### 자동 동작 내용

| 기능 | 설명 |
|------|------|
| **바드 스킬** | 6초 쿨다운으로 디코 또는 프로보 자동 시전 |
| **마스터리 버프** | Inspire + Invigorate (딜 모드) 또는 Resilience + Perseverance (생존 모드) |
| **힐** | 파티원/리더 HP 77% 이하 시 Greater Heal 또는 Arch Cure 자동 시전 |
| **펫 힐** | 펫 HP 77% 이하 시 힐/큐어 자동 시전 |
| **Protection** | Protection 버프 자동 유지 |
| **Doom 타일** | 둠 던전 타일 자동 회피 |

### 바드 모드 선택

- **디스코던스 (기본)**: 몬스터 능력치를 약화. 탱커 중심 파티에 적합.
- **프로보케이션**: 몬스터끼리 싸우도록 유도. 딜러 파티에 적합.

## 7. 리더 명령어 전체 목록

리더 컨트롤 검프의 버튼이 **채팅 명령어**를 자동으로 전송합니다.
리더가 직접 채팅창에 입력해도 동작합니다.

### 공통 명령어 (쓰로잉봇 + 바드봇 모두 반응)

| 명령어 | 효과 |
|--------|------|
| `!정지` | 봇 일시정지 |
| `!재개` | 봇 재개 |
| `!팔로우` | 팔로잉 모드 (전투 억제, 리더 추적만) |
| `!정상화` | 일반 전투 모드로 복귀 |
| `!상하차` | 상하차 모드 대기 (리더가 다음에 컨테이너 시리얼 전달) |
| `!밥줘` | 펫에게 먹이 1개 지급 |
| `!소환` | 펫 소환 (크리스탈 볼 사용) |
| `!송금` | 골드 수집 (송금모드 On인 봇만) |
| `!sell` | 파워스크롤 NPC 판매 |
| `!trash` | Organizer 트래쉬 실행 |
| `!파스정리` | 파스정리 대기 모드 |
| `!분홍정리` | 분홍정리 대기 모드 |
| `!해골정리` | 해골정리 대기 모드 |
| `!게이트` | 게이트 대기 모드 |
| `!헬프` | 탈출 도움 요청 처리 |
| `0x[시리얼]` | 해당 시리얼 적에게 집중 점사 |

### 쓰로잉봇 전용 명령어

| 명령어 | 효과 |
|--------|------|
| `!max` | 원거리 MAX 모드 (12타일 유지) |
| `!선공` | 선공 모드 |
| `!챔프 [봇이름]` | 챔프 구역 지정 |
| `!아머` | Armor Ignore 모드 ON/OFF |
| `!에너미` | Enemy of One 발동 |
| `!에너미오프` | Enemy of One 해제 |
| `!뱀폼` | 뱀파이어폼 시전 |
| `!날아` | 비행 활성화 |
| `!s[번호]` | 룬북 Sacred Journey 이동 (예: `!s3`) |

### 바드봇 전용 명령어

| 명령어 | 효과 |
|--------|------|
| `!프로보` | 프로보케이션 모드로 전환 |
| `!디코` | 디스코던스 모드로 전환 |
| `pf` | Poison Field 시전 (현재 점사 타겟에) |
| `r[번호]` | 룬북 Recall 이동 (예: `r3`) |
| `g[번호]` | 룬북 Gate Travel 이동 (예: `g2`) |

### 상하차 사용 방법

```
1. 리더가 "!상하차" 입력 → 봇이 "상하차 대기" 상태 진입
2. 리더가 컨테이너에 대고 우클릭 → 시리얼 번호 확인
3. 리더가 "0x[시리얼]" 입력 → 봇이 컨테이너로 아이템 이동 시작
```

> 리더 컨트롤의 **상하차** 버튼을 누르면 위 과정이 자동화됩니다.

## 8. GGO 장비최적화도우미 v1.0

보관 중인 장비들을 자동 분석하여 **최적 악세사리 조합**을 찾아주는 도구입니다.
창고나 보관함에 장비를 넣어두고 실행하면, 현재 스탯 기준으로 가장 효율적인 조합을 추천합니다.

> 이 스크립트는 RazorEnhanced에서 실행하면 Windows Forms 별도 창이 열립니다.

### 화면 구성

```
┌─────────────────┬──────────────────────────────────┐
│   왼쪽 패널      │         오른쪽 결과 목록           │
│ ─ 현재 스탯 입력 │  순위 | 조합 요약 | 점수 | 상태    │
│ ─ 목표 스탯 입력 │                                  │
│ ─ 슬롯 선택     │                                  │
│ ─ 확장 검색 설정 │                                  │
│ ─ 스캔 버튼     │  [◀ 이전]   1/5 페이지   [다음 ▶] │
└─────────────────┴──────────────────────────────────┘
```

### 사용 방법 (단계별)

**Step 1. 현재 스탯 입력**
- **★ 현재 스탯 자동 읽기** 버튼을 클릭하면 장착 중인 장비 기준으로 자동 입력됩니다.
- 가고일 캐릭터는 HCI에 **5를 더해서** 수동 수정하세요 (종족 특성).

**Step 2. 목표 스탯 설정**
- 기본값(HCI 50, DI 100, SSI 55 등)을 본인 빌드에 맞게 수정하세요.
- ★ 표시 항목이 핵심 목표치 (이 조건을 충족하는 조합만 표시).

**Step 3. 스캔 슬롯 선택**
- 찾고 싶은 슬롯만 체크하세요.
- 기본값: 악세사리 4종 (Necklace, Ring, Bracelet, Shield) 선택됨.
- 갑옷까지 포함하면 계산량이 크게 늘어납니다.

**Step 4. Antique 설정**
- **Antique 아이템 제외** 체크 시: 내구도 소모 장비는 일반 결과에서 제외.
- Antique 포함 조합은 결과 목록 하단에 별도로 표시됩니다 (주황색).

**Step 5. 확장 검색 (STR/DEX/INT 재분배)**
- 체크 시: STR/DEX/INT 기준치를 조정해서 더 많은 조합을 탐색합니다.
- **리얼 스탯 자동 추정** 버튼으로 장비 보너스를 제외한 실제 능력치를 계산합니다.
- 결과에서 **하늘색** 행 = 스탯 조정이 필요한 조합.

**Step 6. 스캔 실행**
- **컨테이너 스캔 & 최적화** 클릭 → 스캔할 컨테이너를 게임 내에서 클릭.
- 여러 개의 컨테이너를 연속으로 클릭 가능 → ESC로 선택 완료.
- 결과가 점수 순으로 표시됩니다.

**Step 7. 아이템 꺼내기**
- 결과 목록에서 원하는 조합 행을 클릭 후 **★ 선택 조합 → 가방으로 꺼내기** 클릭.
- 해당 조합의 아이템이 백팩으로 이동됩니다.

### 결과 해석

```
예시: [Neck,Ring,Brac,Shie] H52 D25 S58 DI103 FC2 FCR6 STR151 Stam215  (+hp8 mana95 lmc40 lrc100)
                             │    │   │   │     │   │    │      │
                            HCI  DCI SSI  DI    FC  FCR  STR  Stamina
```

- **H**: HCI (Hit Chance Increase)
- **D**: DCI (Defense Chance Increase)
- **S**: SSI (Swing Speed Increase)
- **DI**: Damage Increase
- **FC**: Faster Casting / **FCR**: Faster Cast Recovery
- **[STR+n]**: 이 수치만큼 STR을 올려야 함 (확장 검색 결과)

## 9. 자주 묻는 질문 (FAQ)

**Q. 봇이 리더를 따라가지 않아요.**
→ 설정 마법사에서 리더를 올바르게 선택했는지 확인하세요.
→ 검프의 **재설정** 버튼을 눌러 설정을 다시 입력하세요.

**Q. 게이트를 타지 않아요.**
→ 리더가 게이트 주문을 시전할 때 봇이 주변 5타일 이내에 있어야 합니다.
→ 게이트 포착 후 12초 내에 탑승하지 못하면 타임아웃됩니다.

**Q. 봇 스크립트 시작 시 "설정 초기화 대기" 메시지가 나와요.**
→ 이전에 저장된 리더/파티원이 현재 주변에 없을 때 발생합니다.
→ 파티 구성원이 모두 접속한 후 재시작하거나, 검프의 **재설정**을 누르세요.

**Q. 봇이 죽었을 때 어떻게 되나요?**
→ 유령 상태가 되면 리더를 따라가며 부활 대기합니다.
→ 부활 후 Dress 에이전트(`th` / `luck`)로 장비를 자동 착용합니다.

**Q. 장비최적화도우미에서 결과가 0개 나와요.**
→ 목표 스탯이 너무 높게 설정되어 있을 수 있습니다.
→ HCI, SSI 등 핵심 목표치를 낮춰보세요.
→ 확장 검색(STR 재분배) 옵션을 켜보세요.

**Q. 봇공통_모듈을 찾을 수 없다는 오류가 나와요.**
→ `GGO_봇공통_모듈.py`가 봇 스크립트와 **같은 폴더**에 있는지 확인하세요.
